# A/B 표본 작업 후보 (양식)

> 실제 커밋/PR로 검증 가능한 과거 작업을 고른다(정답=실제 머지본과 대조). 난이도 골고루 20~30건.
> 각 항목에 커밋 해시/PR번호를 채워 재현.

## easy (단일파일·명확·저위험 → 저비용 모델 후보)
- [ ] <UI 문구/속성 소수정>  · ref <commit/PR>
- [ ] <정적 설정 파일 한 줄 수정>  · ref <...>
- [ ] <로그 grep 기반 단순 원인 확인>  (저비용 explorer arm 전용)

## med (다중파일·기존패턴 확장 → 중급 모델 핵심 판정 구간)
- [ ] <기존 컴포넌트에 필드/표시 로직 추가>  · ref <...>
- [ ] <단일 모듈 버그픽스 + 테스트>  · ref <...>
- [ ] <알림/조건 1개 추가>  · ref <...>

## hard (동시성·실시간·도메인로직 → 상위모델 유지 근거 수집용)
- [ ] <시간대/경계 버그 근본수정>  · ref <...>
- [ ] <재연결/락/상태머신 로직>  · ref <...>
- [ ] <스키마/마이그레이션 동반 변경>  · ref <...>

## 채점 시 주의
- hard 항목은 "중급 모델 실패 증거 수집"이 목적 — 여기서 무너지면 레인 C 유지 근거.
- 각 arm 결과는 scoring-template.md 복사해 블라인드(A/B/C) 채점.
