# 라우팅 게이트 (위험플래그 → 모델 승급)

매 위임 전 적용. 자연어 "위험명령" 해석은 모델마다 다르므로 **플래그로 판정**한다.

## 위험플래그

| 플래그 | 예시 |
|---|---|
| 돈/거래 | 주문·결제·정산·잔고 변경 로직 |
| 인증 | 로그인·세션·토큰·권한·OTP |
| 시크릿 | API키·비밀번호·인증서 다루는 코드 |
| 데이터삭제 | DELETE/DROP/TRUNCATE, 대량 삭제 |
| 프로덕션배포 | deploy, 컨테이너 재시작, 라이브 반영 |
| 동시성 | 락·race·병렬·큐·상태머신 |
| 마이그레이션 | 스키마 변경, 데이터 이관 |
| 장애분석 | 라이브 장애 원인추적, 복구 명령 포함 |

## 게이트 규칙

- **0개** → 레인 A/B 자유 (저비용 모델 OK)
- **1개** → 중급 모델↑ + 리뷰(추론기 review) 붙임
- **2개 이상** → 레인 C(상위 모델 직접) 또는 적대적 교차검증(challenge) 강제

이 게이트가 하위 오케스트레이터(예: OMC)의 haiku/sonnet/opus 기본 라우팅보다 **우선**한다.

## 2중 판정 (반박 반영 — 과소판정 방지)

1. **메인이 사전 판정**: 위임 전 위험플래그 체크리스트를 명시(어떤 플래그가 왜 켜졌는지).
2. **하위 에이전트가 시작 시 재판정**: 미션 받으면 스스로 플래그 재평가. **메인 판정과 불일치하면 STOP하고 보고**(과소판정된 작업이 저비용 모델로 흘러가는 걸 막음).

## 교차검증 시 주의

- 위험영역 교차검증은 **redact 전제**(민감맥락 최소 노출). 더 많은 에이전트가 더 많은 시크릿을 보게 만들지 말 것.
- 교차검증이 독립 검증이 되려면 상대의 추론을 주지 말고 **코드/테스트에서 독립 재도출**하게 한다(상관오류 방지).
