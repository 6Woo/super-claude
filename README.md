# super-claude

Claude Code(및 유사 에이전트형 코딩 CLI)에서 **상위 모델은 판단에만 쓰고, 노동은 저비용 모델·외부 추론기에 안전하게 위임**하는 운영 프레임워크.
목표: 토큰 절약 · 생산성 향상 · 에이전트 적극 활용 — 단 **프로덕션 안전을 깨지 않으면서**.

## 핵심 통찰

> 시스템 프롬프트로 이식 가능한 건 **행동 규율**이지 **지능**이 아니다.

- ✅ 이식 되는 것: 도구 사용 판단, 파일 안 읽고 답하는 버릇 교정, 추측 금지, 중도 포기 금지, 검증·완결성
- ❌ 안 되는 것: 추론 깊이, 아키텍처 판단력, 긴 맥락 통합

그래서 하위 모델은 **규율로 조이고**, 지능이 필요한 **판단만 상위 모델**에 남긴다. "상위 모델 복제"가 아니라 "하위 모델이 자기 능력을 100% 규율 있게 쓰게" 만드는 게 목표.

## 3 레인 (리스크 기반 위임)

| 레인 | 작업 | 일꾼 | 규칙 |
|---|---|---|---|
| **A** 조사 | 탐색·분석·리서치·리뷰 (쓰기 0) | 저비용 모델 + 외부 추론기, 병렬 자유 | 리스크 0, 자동 OK — 여기가 토큰절약 핵심 |
| **B** 구현 | 코드 변경 (쓰기) | 중급 모델, 좁은 파일소유권 | **worktree 격리 + 테스트 있을 때만** 병렬. 얽힌 프로덕션은 순차 |
| **C** 판단 | 아키텍처·라이브/DB/인증/보안/배포·고위험 최종리뷰 | 상위 모델, 인라인 | 하위 모델로 안 내림 |

## 왜 이렇게까지 하나 (안전 우선)

외부 추론기가 "공짜 지갑(별도 요금제)"이라도 **맥락 포장·검증·통합 비용은 남는다** → 위임은 **덩어리 크고 독립적일 때만**. 작은 일은 직접이 싸다.
병렬 2배는 **독립성이 높을 때만** 성립 — 라이브 서버는 DB/인증/배포/동시성이 얽혀 병렬 구현 분해 자체가 위험하다. **안전은 프롬프트 문장이 아니라 도구·권한·격리로 강제한다.**

## 구조

```
CLAUDE.md.example          # ~/.claude/CLAUDE.md 에 붙여넣을 팀 오케스트레이션 섹션
protocols/
  delegation-protocol.md   # 위임 전체 절차 + 하위모델 규율 코어 10줄
  routing-gate.md          # 위험플래그 → 모델 승급 게이트 (2중 판정)
  subagent-missions.md     # explorer / worker / reviewer / integrator 미션 템플릿 (영어)
ab-validation/             # 다운그레이드가 품질 유지하는지 '증거로' 검증하는 키트
  README.md                # A/B 절차 + 합격선
  scoring-template.md      # 작업 1건당 블라인드 채점 시트
  tasks.example.md         # 표본 작업 후보 양식
```

## 쓰는 법

1. `CLAUDE.md.example` 섹션을 네 `~/.claude/CLAUDE.md`에 붙여넣기
2. 위임할 때 `protocols/subagent-missions.md` 템플릿의 대괄호만 채워 하위 모델/추론기에 전달
3. 하위 모델 기본 채택 **확정 전** `ab-validation/`으로 실측 (체감 아니라 증거)

## 언어 정책

- **모델용 지시(미션 템플릿)** = 영어: 하위 모델·외부 추론기의 준수율↑ + 토큰↓
- **사람이 읽고 유지하는 문서(README·CLAUDE.md 섹션)** = 사용자 언어: 유지보수성 + 뉘앙스 보존

## 정직한 한계

- 하위 모델이 상위 모델 지능을 갖게 되진 않는다. 규율만 이식된다.
- 얽힌 프로덕션 병렬 구현은 위험 — 레인 B 가드가 없으면 하지 마라.
- "검증 후 통합"이 품질을 보증하진 않는다 — 두 모델이 같은 함정에 같이 빠질 수 있다(상관오류). 최종 판단은 코드·테스트·로그 증거로.

## 설계 검증

이 프레임워크 자체를 외부 추론기(codex/GPT)의 plan·challenge·review 교차검토로 다듬었다. 반박에서 나온 지적(위험판정 주체·공유리소스 격리·commit 책임·라이브명령 경계)을 미션 템플릿에 반영했다.
